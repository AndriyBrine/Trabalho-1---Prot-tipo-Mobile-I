# Trabalho-1-Protipo-Mobile-I
Repositório do trabalho incluindo: Print das telas do Figma, Link do protótipo do Figma, SQL do modelo de dados, PDF do MER e PDF da apresentação.

Objetivo do Trabalho: Desenvolver um protótipo de interface para
um sistema mobile utilizando o Figma e o modelo de entidade
relacionamento.

Requisitos:
1. Interface com o usuário: O trabalho entregue está com a interface amigável: identidade visual adequada, alinhamento, boa tipografia, capricho, hierarquia de informação visual, paleta de cores propriada, imagens com boa resolução, navegabilidade, aplica leis de proximidade da Gestalt, lei de semelhança, lei de continuidade, lei de área comum, Lei de área clicável (Lei de Fitts) etc.
2. Conteúdo:
  > O segmento de negócio escolhido é descrito em alguma tela do protótipo;
  > As telas possuem textos bem claros e imagens informando o objetivo do segmento de negócio;
  > Todas as telas do protótipo foram criadas com conteúdo adequado;
  > Escolha um logo e um nome para o sistema. O nome e o logo devem ser visíveis no sistema.
4. Funcionalidade:
  > O sistema apresentou no mínimo 5 telas;
  > O sistema apresentou tela de autenticação do usuário;
  > O sistema apresentou a tela de criação do usuário;
  > O sistema apresentou a tela de recuperação da senha.
5. Github:
  > Print com as telas do Figma
  > Link do protótipo do Figma
  > Incluir no Github o SQL do modelo de dados.
  > Incluir no Github o pdf do modelo de entidade e relacionamento
  > Incluir no Github o pdf da apresentação do trabalho
  > Repositório do trabalho e incluindo todos os arquivos de forma organizada
6. Modelo de Entidade Relacionamento:
  > As entidades representam corretamente os elementos do mundo real?
  > Há no mínimo 3 entidades no modelo?
  > Os relacionamentos refletem corretamente as associações entre as entidades?
  > Os atributos são apropriados e bem distribuídos entre entidades e relacionamentos?
  > As cardinalidades (1:1, 1:N, N:M) estão bem definidas e fazem sentido no contexto do sistema?
  > As restrições de integridade garantem consistência (chaves primárias e estrangeiras corretamente definidas)?
  > O modelo evita redundância de dados?
  > Há dependências funcionais inadequadas que poderiam ser normalizadas?
  > O MER cobre todos os requisitos do sistema conforme especificado?
  > Os relacionamentos refletem a lógica de negócios corretamente?
7. Slides de Apresentação de Trabalho:
  > A apresentação deve ser entregue no formato PDF.
Especificação dos elementos que devem constar nos slides de
apresentação do trabalho:
    1) Capa - Incluir nome, logotipo, IFES, nome dos integrantes e data.
    2) Justificativa - Explicação dos motivos da escolha do tema e como nasceu a ideia.
    3) Objetivo - Descrever o objetivo geral do trabalho. Um bom objetivo deve ser verificável ao final do trabalho.
    4) Funcionalidades - Descrever em tópicos todas as funcionalidades criadas no protótipo.
    5) Protótipos - Incluir prints de telas do App, bem como uma breve descrição
    6) Modelo de Entidade e Relacionamento
    7) Link do Github
8. Apresentação e Avaliação:
  > Todos os integrantes estavam presentes na apresentação  O grupo explicou o código e as telas com precisão e sem dúvidas.
  > O grupo respondeu a todas as perguntas da professora sem nenhuma dúvida.
